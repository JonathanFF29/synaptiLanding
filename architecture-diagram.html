02 · AWS Ingest & Trigger
☁️ Amazon Web Services — Storage & Event Layer
S3
Storage
Amazon S3
Input bucket receives the raw video file. Eleven-nine durability. An S3 Event Notification fires the moment the upload is complete — no polling, no manual step.
input/ output/ reports/
λ
Orchestration · Trigger
AWS Lambda — Ingest Handler
S3 event wakes this serverless function. It validates the file, writes a job record to the database, then enqueues a message in SQS. No servers to maintain.
event-driven serverless
SQS
Job Queue
Amazon SQS
Durable FIFO queue that decouples ingest from processing. Handles traffic bursts, retries failed jobs automatically. No video is ever lost or skipped.
FIFO auto-retry DLQ
⚡ Event bus: S3 upload → Lambda (ingest) → SQS message → AWS Batch job submission — fully automatic, no manual steps
↓
03 · GPU Compute — The Worker
⚙️ AWS Batch + EC2 GPU — Inference Pipeline
BCH
Batch Processing
AWS Batch
Reads SQS messages and provisions GPU capacity per job. Scales to dozens of simultaneous videos. Releases instances as soon as the job completes — zero idle machines.
auto-scale spot + on-demand
GPU
Inference Engine
EC2 · NVIDIA GPU (G4dn / G5)
NVIDIA T4 (G4dn) or A10G (G5) cards. Runs the detection models at full resolution. Hours of footage processed in minutes. Container-based — Docker image pulled from ECR.
NVIDIA T4 NVIDIA A10G CUDA
🔬 Worker: AI Pipeline — 5 Stages executed per video
STEP 01
🎥

Ingest

Worker pulls the raw video from S3. Decodes frames with FFMPEG. Validates resolution and framerate.

→
STEP 02
🔍

Detection

Every frame runs through our computer vision model. Detects & classifies: moto, car, truck, bus, bicycle, pedestrian.

→
STEP 03
🎯

Tracking

Assigns a persistent unique ID to each object. Follows vehicles across occlusions. Eliminates double-counts.

→
STEP 04
🗺️

Origin-Destination

Virtual lines define zones. Each tracked vehicle is mapped to an O→D pair. Counts accumulated per route & class.

→
STEP 05
📊

Aggregate

Worker writes results: OD matrix, per-class counts, route stats, per-vehicle track IDs. Pushes outputs to S3.

🚀 GPU Worker also writes an annotated video (bounding boxes, IDs, route lines) back to S3 output/ — alongside the raw data files.
↓
